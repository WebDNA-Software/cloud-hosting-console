Each firewall is an asset (type=firewall, provider_id = firewall id). Nested rule arrays (rules.inbound[] / rules.outbound[]) have no [results] count of their own, so we iterate a BOUNDED loop (Linode caps rules at 25/direction) and only count/render an entry when its action is ACCEPT or DROP via [switch]; absent entries resolve to a literal [..] that matches neither, so they're skipped. Outer (firewall) loop [index] is captured to [fwi] before the inner (rule) loop, since [index] always refers to the innermost loop.
