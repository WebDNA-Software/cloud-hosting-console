Two layers, both API-driven

The managed Backups service runs automatic rotating backups (daily window + weekly + the most recent), restorable to a new or existing Linode. Snapshots are a manual point-in-time capture you trigger before risky changes — they don't rotate out.

For the reseller model, off-server copies belong in Object Storage — the durable, cross-region layer the per-VM backup service doesn't replace.

Restore flow
  1. Pick a backup or snapshot for the Linode.
  2. Restore in place, or to a brand-new Linode in either AU region.
  3. Reattach firewall + VPC; the DNS A record can repoint if the IP changes.
